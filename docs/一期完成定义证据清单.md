# QQ 群机器人一期完成定义证据清单

本文用于把 `一期产品蓝图.md` 的完成定义拆成可留存证据。它不替代真实验收；本地审计只能证明代码、测试、后台入口和文档已经具备，真实 QQ 群运行和真实恢复仍需要单独留证。

## 1. 本地可自动审计证据

执行：

```bash
python scripts/phase_one_audit.py --report-file data/phase-one-audit.json
```

预期：

- `failed=0`
- `required` 全部通过
- `manual_required=2`
- 状态为 `passed_with_manual_gates`

说明：

- 该脚本只读检查源码、测试、后台页面、运维脚本和文档。
- 不连接 QQ、不连接数据库、不读取 `.env`、不执行恢复。
- `manual_required` 用来提示仍需真实群运行和真实恢复演练证据。

## 3. 手动证据门槛

### 3.1 真实群持续运行一周

至少保留：

- 运行周期的起止时间。
- `docker compose ps` 或健康检查截图/输出。
- 后台消息日志、回复日志、LLM 调用日志。
- OneBot/NapCat 重连或异常恢复记录。
- 目标群内核心功能测试记录。

### 3.2 真实恢复演练

至少保留：

- `sh scripts/backup.sh` 输出和备份文件路径。
- `FORCE=1 sh scripts/restore_postgres.sh <备份文件>` 输出。
- 恢复后的 `scripts/smoke_check.py` 输出。
- 恢复后后台可登录、群配置和业务数据仍存在的截图/记录。

## 4. 不能用本地审计替代的内容

- 真实 QQ 群消息收发。
- QQ 小号离线、重连和风控后的恢复。
- 生产数据库真实恢复。
- 一周持续运行稳定性。

只有本地审计通过、真实群一周运行证据齐全、真实恢复演练完成后，才可以把一期标记为完整完成。
