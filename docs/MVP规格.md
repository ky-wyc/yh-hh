# QQ 群机器人 MVP 规格

## 1. 阶段目标

MVP 只验证最小闭环：QQ 小号能稳定接入，Bot Core 能收到消息、判断是否响应、调用大模型、发送回复，并能通过后台完成最小配置和排障。

MVP 不追求完整产品体验，不实现长期记忆、知识库、每日总结、复杂游戏、自动群管和复杂图表。

## 2. 交付范围

### 2.1 QQ 接入

- 使用 OneBot v11。
- MVP 固定采用反向 WebSocket。
- Bot Core 暴露 `/onebot/ws`。
- OneBot 实现主动连接 Bot Core。
- 支持接收群文本消息。
- 支持向群发送文本消息。
- 支持断线自动重连和连接状态记录。

验收：

- Linux 服务器上可完成 QQ 小号登录。
- Bot Core 后台能看到 OneBot 在线状态。
- 断开 OneBot 连接后可恢复连接。
- 指定 QQ 群里发送文本消息，Bot Core 能记录事件。

### 2.2 消息路由

- 支持事件标准化为内部 `BotEvent`。
- 支持基于 `dedup_key` 的消息去重。
- 支持群白名单。
- 支持 `/help`。
- 支持 `/ping`。
- 支持 `@机器人`。
- 支持 `/ai 问题`。
- 支持被丢弃消息的原因记录。

验收：

- 白名单外群不响应。
- `/ping` 返回固定响应，不调用大模型。
- 重复事件不会重复回复。
- 后台或日志能看到消息处理结果。

### 2.3 大模型调用

- 支持 OpenAI-compatible 接口。
- 支持配置 `base_url`、`api_key`、`model`、`temperature`、`max_tokens`、`timeout`。
- 未配置 API Key 时返回明确错误。
- 模型失败时返回短错误，不刷屏。
- 记录调用日志和耗时。

验收：

- `/ai` 可得到模型回复。
- `@机器人` 可得到模型回复。
- 错误配置不会导致 Bot Core 崩溃。
- 大模型调用日志可查询。

### 2.4 最小 Skill

- `help`：展示可用命令。
- `ping`：健康响应。
- `ai`：大模型问答。
- `dice`：掷骰子。
- `admin-lite`：管理员手动警告和关键词拦截。

验收：

- `/dice` 不调用大模型。
- 普通用户不能执行管理员命令。
- 管理员警告操作写入日志。
- 关键词命中可按配置拦截或提示。

### 2.5 数据存储

- PostgreSQL 保存群、用户、消息摘要、回复记录、配置和日志。
- Redis 保存短期上下文、限流状态和临时锁。
- 不在 MVP 实现知识库向量检索业务，但数据库可预留 pgvector 扩展。

验收：

- 重启 Bot Core 后群配置仍存在。
- 消息摘要可查询。
- Redis 故障时有明确错误，不静默丢失状态。

### 2.6 Web 后台

- 登录。
- 查看 OneBot 连接状态。
- 查看 Bot 运行状态。
- 配置群启用开关。
- 配置默认回复模式。
- 配置大模型参数。
- 查看消息日志。
- 查看系统错误日志。

验收：

- 后台必须登录后访问。
- API Key 不明文展示。
- 写操作有权限校验。
- 可通过后台完成一次模型配置并测试。

### 2.7 部署

- Docker Compose 启动 `bot-app`、`onebot`、`postgres`、`redis`。
- MVP 不启动 Qdrant。
- 提供 `.env.example`。
- 提供健康检查接口。
- 提供最小备份脚本或备份命令说明。

验收：

- `docker compose up -d` 后核心服务可启动。
- 容器异常退出后可自动重启。
- PostgreSQL 数据目录持久化。
- 健康检查接口可访问。

## 3. 明确不做

这些能力不属于 MVP，但已经在后续阶段安排：

- 长期记忆：一期。
- 知识库问答：一期。
- 每日群聊总结：一期。
- `/guess` 和持久游戏状态：一期。
- 自动禁言、踢人和模型判定群管：一期后段或二期。
- 文件知识库、PDF、Word 解析：二期。
- Qdrant 独立向量库：二期。
- 群聊情绪、风险分析和复杂图表：二期。
- 多模型路由和成本精细统计：二期。

## 4. 完成定义

MVP 完成必须同时满足：

- 指定 QQ 群能完成接收消息和发送回复闭环。
- `/help`、`/ping`、`/ai`、`/dice` 可用。
- 后台可登录、配置群开关、配置模型、查看日志。
- 数据可持久化，服务可通过 Docker Compose 恢复。
- OneBot 接入风险、账号登录恢复流程和常见故障处理有文档说明。
