# QQ 群机器人 MVP 运维说明

## OneBot 登录恢复

MVP 默认使用 Lagrange.OneBot 作为 Linux 云端接入实现。OneBot 容器的数据目录挂载到：

```text
./data/onebot
```

该目录用于保存配置和登录态。更换服务器或恢复服务时，应先恢复该目录，再启动 `onebot` 容器。

本仓库提供了一个反向 WebSocket 配置样例：

```text
deploy/lagrange/appsettings.example.json
```

部署时可复制到：

```text
data/onebot/appsettings.json
```

然后按实际 QQ 小号和 Lagrange.OneBot 版本调整 `Account`、登录方式和 `Implementations`。Lagrange.OneBot 的具体配置字段以官方文档为准：https://lagrangedev.github.io/Lagrange.Doc/v1/Lagrange.OneBot/

## 反向 WebSocket

Bot Core 暴露：

```text
ws://bot-app:8000/onebot/ws
```

OneBot 实现需要配置为主动连接该地址。若配置了 `ONEBOT_ACCESS_TOKEN`，OneBot 请求需要携带：

```text
Authorization: Bearer <token>
```

## 常见故障

### 后台显示 OneBot 离线

检查项：

- `bot-app` 是否运行。
- `onebot` 是否运行。
- OneBot 反向 WebSocket 地址是否为 `ws://bot-app:8000/onebot/ws`。
- OneBot 登录态是否过期。
- `ONEBOT_ACCESS_TOKEN` 是否一致。

### 群里没有回复

检查项：

- 群是否在 `ALLOWED_GROUPS` 中。
- 后台群开关是否启用。
- 回复模式是否允许命令或 @ 回复。
- 是否触发了限流。
- `/api/system/logs` 中是否有丢弃原因。

### 大模型不可用

检查项：

- 后台是否配置 API Key。
- `LLM_BASE_URL` 是否是 OpenAI-compatible 接口。
- 模型名是否正确。
- `/api/settings/llm/test` 是否返回失败原因。
