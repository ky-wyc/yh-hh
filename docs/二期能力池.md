# QQ 群机器人二期能力池

## 1. 阶段定位

二期不再是单一闭环建设，而是增强规模、智能度、运营效率和可扩展性。二期能力应按实际使用反馈排序，不建议一次性全部实现。

## 2. 能力分组

### 2.1 知识库增强

能力：

- Markdown 上传。
- TXT 上传。
- PDF 解析。
- Word 解析。
- 网页导入。
- 文档重新向量化。
- 文档版本管理。
- Qdrant 独立向量库。
- pgvector 和 Qdrant 可切换。

前置条件：

- 一期知识库最小版本稳定。
- 文档分块和引用格式已经被实际验证。

验收方向：

- 大文件处理不会阻塞 Bot 消息响应。
- 向量化失败可重试。
- Qdrant 切换不影响上层问答接口。

### 2.2 记忆增强

能力：

- 自动记忆提取。
- 记忆压缩。
- 记忆冲突检测。
- 记忆过期策略。
- 用户查看和删除自己的记忆。

前置条件：

- 一期手动记忆和审核流程稳定。

验收方向：

- 自动提取默认进入 pending。
- 敏感身份、隐私和攻击性标签不得自动写入 approved 记忆。
- 记忆压缩前后可追溯来源。

### 2.3 群聊分析

能力：

- 今日消息数。
- 活跃成员。
- 热门话题。
- 热门关键词。
- 未回答问题。
- 情绪趋势。
- 争吵风险。
- 广告风险。
- 后台图表分析。

前置条件：

- 消息摘要保存策略稳定。
- 群隐私和保留周期已配置。

验收方向：

- 分析结果不泄露其他群数据。
- 风险判断默认只提醒管理员，不直接处罚用户。

### 2.4 群管规则引擎

能力：

- 更完整的规则配置。
- 多条件组合。
- 命中后动作链。
- 二次确认。
- 操作回滚。
- 黑名单和白名单。
- 自动禁言策略。
- 踢人策略。

前置条件：

- 一期基础群管审计稳定。

验收方向：

- 高风险动作必须有权限、原因和审计。
- 模型判定不能绕过规则引擎直接执行。
- 规则变更可追溯。

### 2.5 娱乐和活动

能力：

- 词语接龙。
- 故事接龙。
- 文字战斗。
- 抽签。
- 群活动报名。
- 活动提醒。

前置条件：

- 一期游戏状态管理稳定。

验收方向：

- 游戏状态按群隔离。
- 游戏不会频繁触发大模型导致成本失控。
- 管理员可中止异常活动。

### 2.6 模型和成本治理

能力：

- 多模型路由。
- 备用模型。
- 高成本模型权限控制。
- 按群、用户、Skill 统计 token。
- 成本估算。
- 额度预警。
- 模型调用质量记录。

前置条件：

- MVP 或一期 LLM 调用日志完整。

验收方向：

- 达到额度时能降级。
- 失败重试不会刷屏。
- 不同模型配置可回滚。

### 2.7 插件化增强

能力：

- Skill 元数据注册。
- Skill 配置 schema。
- Skill 调用统计。
- 插件市场式管理。
- Skill 开发模板。

前置条件：

- 一期 Skill 开关和权限稳定。

验收方向：

- 新 Skill 不需要改动消息路由主流程。
- Skill 禁用后不会继续响应。
- Skill 错误不会拖垮 Bot Core。

## 3. 二期优先级建议

建议按以下顺序挑选：

1. 知识库文件上传和解析。
2. 群聊分析和后台图表。
3. 模型成本统计和额度治理。
4. 自动记忆提取和记忆压缩。
5. 群管规则引擎。
6. 复杂娱乐玩法。
7. 插件市场式 Skill 管理。

## 4. 二期完成定义

二期不是一次性完成的阶段。每个能力进入开发前，应先补独立需求说明和验收标准，并确认不会破坏 MVP 和一期已经稳定的消息收发、权限、审计和后台配置闭环。
