# QQ 群机器人一期产品蓝图

## 1. 阶段定位

一期从 MVP 后开始，目标是把机器人从“能跑通闭环”推进到“可长期运营”。这一阶段补齐记忆、知识库、每日总结、基础定时任务、持久游戏状态、审计和更完整的后台管理。

## 2. 前置条件

- MVP 已完成。
- OneBot 接入稳定，断线恢复路径明确。
- Bot Core 已具备事件标准化、消息去重、权限检查和基础日志。
- 后台已能登录并配置模型和群开关。

## 3. 一期里程碑

### 3.1 记忆最小版本

交付：

- 手动记住。
- 手动忘记。
- 后台新增、修改、删除记忆。
- 记忆来源、可信度、审核状态。
- 默认只使用 approved 记忆。

验收：

- 用户明确要求“记住”后可形成记忆。
- 被删除记忆不再参与回答。
- 待审核记忆不会默认参与回答。
- 涉及敏感个人信息的自动写入默认关闭。

### 3.2 知识库最小版本

交付：

- 后台录入纯文本知识。
- 文本分块。
- pgvector 向量检索。
- 群级知识库隔离。
- `/kb` 或知识库问答入口。
- 回答可附来源标题或片段编号。

验收：

- A 群知识库不会影响 B 群回答。
- 知识库没有依据时明确说明未找到。
- 重新向量化失败时可看到错误。
- 后台可检索测试。

### 3.3 每日总结和定时任务

交付：

- APScheduler 或等价调度模块。
- 一次性提醒。
- 每日群聊总结。
- 过期上下文清理。
- 过期游戏状态清理。
- 任务执行历史。

验收：

- 每日总结可按群开启或关闭。
- 任务失败有记录。
- 重启后任务不会重复执行。
- 定时任务不会绕过群开关和权限策略。

### 3.4 持久游戏状态

交付：

- `/guess start`。
- `/guess stop`。
- 游戏状态按群隔离。
- 游戏状态过期清理。
- 一个群同一时间默认只允许一个主要游戏。

验收：

- 不同群的游戏状态互不影响。
- 重启后未过期游戏可恢复。
- 游戏命令不调用大模型。

### 3.5 后台一期增强

交付：

- 群管理详情。
- Skill 开关。
- 长期记忆管理。
- 知识库管理。
- 定时任务管理。
- 群管审计日志。
- LLM 调用日志。

验收：

- 后台可以完成记忆和知识库的基本运营。
- 高风险配置变更写入审计日志。
- API Key 等敏感字段脱敏。

### 3.6 基础群管增强

交付：

- 警告。
- 临时禁言。
- 关键词拦截。
- 刷屏阈值。
- 新人欢迎。
- 手动解除禁言。

验收：

- 普通用户不能执行群管命令。
- 管理员操作必须有原因和审计记录。
- 模型只提供建议，不直接踢人或长时间禁言。

## 4. 一期不做

- 文件知识库上传和 PDF、Word 解析。
- Qdrant 独立向量库。
- 自动记忆提取。
- 复杂情绪和风险分析。
- 插件市场。
- 多模型路由。
- 操作回滚。
- 完整移动端后台。

## 5. 一期完成定义

- 机器人能在至少一个目标群持续运行一周。
- 管理员可通过后台维护群配置、记忆、知识库和定时任务。
- 记忆、知识库、每日总结、基础群管都有审计或日志。
- 数据备份和恢复流程经过一次演练。
