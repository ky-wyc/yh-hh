# QQ 群机器人一期开发状态

## 当前进度

一期目标是从 MVP 进入可长期运营阶段，范围以 `一期产品蓝图.md` 为准。

## 已完成

### 记忆最小版本，基础版本完成

- 已新增长期记忆数据表。
- 已支持 `/remember 内容` 创建待审核记忆。
- 已支持 `/forget 记忆ID` 软删除自己的记忆。
- 已支持后台新增、修改、审核、删除记忆。
- 已支持后台从现有群列表中选择长期记忆作用群，留空则为全局记忆。
- 已支持记忆来源、可信度、审核状态。
- AI 回复默认只读取 `approved` 记忆。
- `pending`、`rejected`、`deleted` 记忆不会参与 AI 回答。
- 记忆新增、修改、删除写入审计日志。

### 知识库最小版本，基础版本完成

- 已新增知识库文档表和分块表。
- 已支持后台录入纯文本知识。
- 已支持文本自动分块。
- 已支持确定性文本向量化并写入 `embedding_json`。
- PostgreSQL 环境中可用 pgvector 时会自动尝试创建扩展和 `embedding_vector` 列。
- 已支持后台配置知识库 Embedding：默认本地确定性向量，也可切换 OpenAI-compatible embedding API。
- 已支持 Embedding API Key 脱敏、清空和后台测试。
- 知识库重建索引会使用当前 Embedding 配置；外部向量化失败时会写入 `index_status=failed` 和 `index_error`。
- 当外部 Embedding 查询临时失败时，知识库检索会保留关键词分数兜底，避免直接中断查询。
- 已支持知识库作用域：全局或指定群。
- 已支持后台新增、修改、启停、删除知识库文档。
- 已支持后台导入 `.txt`、`.md`、`.csv`、`.xlsx`、`.xlsm` 文件；Excel 会按工作表和行区间拆分成多篇知识库文档，避免大文件被截断。
- 已支持导入时从现有群中多选作用群，并为每个选中群生成对应知识库文档；导入完成后返回导入报告。
- 已支持导入时保存原始文件，并在知识库文档中记录来源文件和来源位置。
- 已支持导入或新增文档时生成结构化摘要、关键词和可回答问题，作为 AI 目录索引保存到知识库文档。
- 已支持在后台知识库列表查看 AI 目录状态、结构化摘要和关键词，并可对单篇文档手动“重建目录”。
- 已增强无 Embedding 场景下的关键词检索：完整关键词、编号、英文数字标识和标题命中会获得更高权重。
- 已增强无 Embedding 场景下的目录优先检索：查询会先参考文档摘要、关键词、问题和来源位置，再回到分块片段排序。
- 已支持后台检索测试。
- 已支持后台手动重建知识库索引。
- 已支持后台按群号和索引状态筛选知识库文档。
- 已支持清空最近索引记录。
- 已支持后台批量重建当前筛选范围内的已启用文档索引。
- 已支持后台一键重试失败索引，并返回总数、成功数、失败数和跳过数。
- 已支持把知识库批量重建和失败重试加入后台队列，由调度器异步执行并记录执行历史。
- 已支持后台查看索引错误信息。
- 已支持后台查看最近知识库索引、批量重建和失败重试历史。
- 已支持 `/kb 问题` 查询知识库。
- `/kb` 会优先检索知识库目录和片段；大模型可用时基于命中的片段组织回答，大模型不可用时返回来源标题和片段编号。
- 已验证 A 群知识库不会命中 B 群知识库。

### 工程化能力，部分完成

- 已增加轻量数据库迁移记录表 `schema_migrations`。
- 服务启动时会记录已执行迁移。
- 已支持对旧表进行幂等字段修补，为后续表结构演进预留路径。
- 已支持旧库自动补建一期新增表，云端拉取新版本后可随服务启动迁移。

### 定时任务和每日总结，基础版本完成

- 已新增定时任务表和任务执行历史表。
- 已支持后台新增、编辑、启停、删除定时任务。
- 已支持后台查看任务执行历史。
- 已支持一次性群提醒。
- 已支持基础每日群聊总结，基于近期消息日志生成消息数和最近片段。
- 已支持定时聊天记忆总结任务，周期性读取近期群聊并生成 `pending` 长期记忆，等待后台审核后才会参与 AI 回复。
- 已支持定时清理短期上下文缓存。
- 定时群提醒和每日总结会检查目标群是否启用，避免绕过后台群开关。

### 后台一期增强，部分完成

- 已支持全局 Skill 开关。
- 已支持群级 Skill 开关。
- 已新增独立左侧导航页“功能管理”，集中展示 Skill 分类、命令、风险等级、LLM/知识库/记忆依赖、私聊支持和全局/群级开关。
- 已支持仪表盘展示 OneBot 连接状态、活动状态、在线/离线时长和恢复提示。
- 已支持后台群详情抽屉，展示消息、回复、记忆、知识库、关键词和定时任务统计。
- 已支持高风险 Skill 开关变更写入审计日志。
- 消息路由已接入 Skill 开关：AI、知识库、记忆、娱乐、基础群管等功能可被后台关闭。
- `/help` 会根据当前群实际启用的 Skill 展示可用命令。
- 已支持 OneBot 私聊消息接入；私聊默认关闭，需在后台开启并配置私聊白名单 QQ 后才会响应。
- 私聊支持帮助、健康检查、AI 问答、知识库、掷骰子和长期记忆等安全功能；群管和按群持久游戏不会在私聊中执行。
- 已新增生图 Skill，可通过 `/image` 或 `/draw` 调用独立生图模型配置，适配 OpenAI 兼容中转站 `images/generations` 接口。
- 已新增按群消息数量触发的长期记忆总结，达到后台配置阈值后自动生成 `pending` 记忆，审批后才进入上下文。
- Excel 知识库导入上限调整为 25MB，并按约 50 行分片，降低大表导入不完整和关键词命中不准的问题。

### 联网搜索 Skill，基础版本完成

- 已新增独立 `web-search` Skill，可在后台“功能管理”中做全局和群级开关。
- 已支持 `/search`、`/web`、`/联网`、`/搜索` 显式联网搜索。
- 已支持后台“模型与 API 设置”中配置联网搜索 Provider、Base URL、API Key、结果数量、超时和测试查询。
- 已支持 `searxng` 和 `tavily` 两类搜索服务。
- 已支持自动判断是否需要联网：`/ai`、@ 回复、主动疑问回复和私聊自然对话命中实时类关键词时，会先搜索再交给大模型回答。
- 自动联网默认需要同时打开 `WEB_SEARCH_ENABLED` 和 `WEB_SEARCH_AUTO_ENABLED`，并且 `web-search` Skill 未被后台关闭。
- 搜索 API Key 只保存不回显，不会进入前端响应或日志摘要。

### 持久游戏状态，基础版本完成

- 已新增游戏状态表 `game_states`。
- 已支持 `/guess start` 启动猜数字游戏。
- 已支持 `/guess 数字` 参与猜数字游戏。
- 已支持 `/guess stop` 结束猜数字游戏。
- 游戏状态按群隔离。
- 同一群同一时间只允许一个活跃猜数字游戏。
- 未过期游戏状态落库保存，服务重启后可继续。
- 过期游戏状态可由定时清理任务标记为 `expired`。
- 猜数字命令不调用大模型。
- 后台群详情已展示活跃游戏数量。

### 基础群管增强，基础版本完成

- 已支持 `/mute @用户 秒数 原因` 临时禁言。
- 已支持 `/unmute @用户 原因` 手动解除禁言。
- 已支持后台群详情中配置新人欢迎开关和欢迎语。
- 已支持 OneBot `group_increase` 事件触发新人欢迎。
- 已支持后台群详情中配置刷屏检测阈值、窗口时间和禁言秒数。
- 已支持后台配置违规累计窗口、阶梯处罚开关、处罚倍率和禁言封顶。
- 已支持后台群详情一键套用群管策略模板：观察模式、宽松、标准和严格。
- 已支持普通用户刷屏时自动临时禁言。
- 已支持基于近期 `warn`、`mute`、`flood_mute` 审计记录累计用户违规次数。
- 已支持刷屏重复违规按累计次数自动提升禁言时长，并保留禁言封顶。
- 已支持后台群详情展示近期违规用户、累计次数和最近时间。
- 已支持日志中心按群号、操作、目标类型、目标 ID 和条数筛选审计日志，用于查询违规历史。
- `/warn` 可记录目标用户近期累计违规次数。
- 群管命令、自动刷屏禁言、新人欢迎均写入审计日志。
- 普通用户不能执行禁言、解禁等管理员命令。

### 备份和恢复演练，工具与记录模板完成

- 已支持 `scripts/backup.sh` 生成 PostgreSQL SQL 备份。
- 备份文件生成后会检查非空，并尽量设置为 `600` 权限。
- 已支持 `scripts/restore_postgres.sh` 从 SQL 备份恢复 PostgreSQL。
- 已新增 `scripts/backup_restore_drill.py` 生成非破坏性备份恢复演练报告。
- 演练报告会检查 `.env`、备份文件、PostgreSQL 标识符、NapCat 登录态目录和恢复命令。
- 演练报告明确标记 `restore_executed=false`，避免把准备检查误认为真实恢复。
- 运维文档已补充演练步骤和证据留存清单。

### 一期完成定义审计，工具与清单完成

- 已新增 `scripts/phase_one_audit.py`，用于静态检查一期代码、测试、后台页面、运维脚本、文档和敏感文件跟踪状态。
- 审计脚本不连接 QQ、不连接数据库、不读取 `.env`、不执行恢复，适合本地、CI 和云端重复运行。
- 审计报告会把“真实群持续运行一周”和“真实恢复演练证据”标记为 `manual_required`，避免把本地代码证据误判为完整上线完成。
- 已新增 `docs/一期完成定义证据清单.md`，把一期完成定义拆成本地自动证据、云端真实证据和不能由脚本替代的内容。
